# 版本历史

## v2.2 (2025-11-14)
- 修复“从数据源导入”无法识别大小写与省略 `pre-` 的前缀问题
- 取消对项目名的前缀筛选：只要命中差分规则并且连接符紧邻即可导入
- 代号生成时：先尝试移除预设前缀 `pre-shoot-`/`shoot-`/`pre-kol-`/`kol-`（不区分大小写），若未命中则保留原内容作为代号
- 规则匹配仍按“版本名全称”最长优先且不区分大小写，连接符仅支持 `+`/`-`
- 修复差分规则表右键记忆库菜单列索引错位（列3/4/5：全称/缩写/语言）
- 新增全局快捷键：`Delete` 删除当前焦点表的选中行
- 删除/撤销后重新编号与居中对齐 `#` 列，统一视觉样式
- 右键菜单新增：差分号（列1）和连接符（列2）的常用项与历史记录，连接符包含默认 `+`/`-`

## v2.1 (2025-11-14)
- 手动检查更新改为自动下载并应用更新
- 版本比较采用语义化规则，避免同版本误报
- 打包版更新策略调整为跳转 GitHub 下载
- 新增 Windows 单文件打包 `dist/AutoRenamer.exe`

## v2.0 (2025-11-06)
- **版本号管理重构**: 将版本号统一存储在 `app_logic.py` 中，解决了版本号不一致的问题。
- **更新逻辑修复**: 修复了主程序中“检查更新”按钮的功能，使其能正确获取当前版本号。
- **启动器优化**: 启动器现在会动态读取主程序的版本号，确保更新检查的准确性。

## v1.42 (2025-11-06)
- 添加启动时自动检查更新功能
- 当有新版本时会弹出提示框显示更新内容
- 优化更新检查逻辑,失败时静默处理不影响程序启动

## v1.41
